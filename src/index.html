BaP Framework

The idea is not to use any framework (JS, CSS, or whatever), just HTML, CSS, and Vanilla Javascript. The main concept is to test new ideas quickly and easily.

It is not intended to be a full-featured framework, but a simple tool to help you develop your ideas. However, it is a robust and scalable framework that can be used to develop simple applications and perform even in production environments.


It is important to read the README file before make any change.

Support

This framework is developed to be supported by Google Firebase.
  • Hosting: Firebase Hosting
  • Database: Firebase Realtime Database
  • Analytics: Firebase Analytics
  • Auth: Firebase Auth
  • Recaptcha: Firebase AppCheck / Google cloud reCAPTCHA
You must to create a Firebase project and configure these values in the `src/_main/constants.js` file, and set de CONSTANT.FIREBASE_AVAILABLE with 'true' value.

Highlights

  • I18N support
  • Storage control
  • URL path router control

Custom component

bap-header

Standard header to use in every page 

<bap-header></bap-header>

bap-footer

Standard footer to use in every page 

<bap-footer></bap-footer>

bap-loading-state

A full screen message intended to notify user the web is processing. You can add it at the beggining of the HTML file to show it before content is loaded, and finally remove the component from the DOM. 

<bap-loading-state sub-message="Sub message">Main message</bap-loading-state>

bap-spinner

A simple spinner animated to use it as you want. 

<bap-spinner></bap-spinner>

bap-svg-image

Wrapper to standarize the SVG image import 

<bap-svg-image name="chevronLeft" size="x3"></bap-svg-imager>

bap-notification

Custom tool to show user messages using Toasts or Snackbars. This component works by request using javascript, so you have to import the main function to make it works: 

import { bapNotify } from "./_main/util.js";

You can see an example in the JS file asociated to this HTML.

Develop and Deploy

Optimizing the code

In order to ensure your code, it was implemented a GULP file to:

  1. Apply I18N to HTML files including in the gulpfile: function replacingHeadMetatags() -> const pagesToProcess
  2. Minify HTML files
  3. Minify CSS files
  4. Minify and obfuscate JS files
  5. Move files to /public folder

To execute correctly, before any deploy you must execute this instruction in your code folder terminal

npm run optimize

Deploying pre production environment

npm run deploy-qa

Firebase modify the URL each couple of day, so you have to keep updated CONSTANT.ENV.URL.QA before deploy


Deploying production environment

npm run deploy-prod

You have to set CONSTANT.ENV.URL.PROD before deploy
